**Goal:** Ship the end-to-end security handshake so a paired mobile client can open a single WSS session to the local relay, with Ed25519-signed `ClientHello` + 6-digit one-time pairing codes + nonce-replay defense + revoke CLI. Plumbs W1.4's `ApprovalBridge` into an actual WebSocket.

**Architecture:** Ports & Adapters per `openspec/project.md`. Crypto in `shared/core/crypto/` (Ed25519 via expect/actual — BouncyCastle on JVM+Android, iOS stub until W2.1). Pairing identity in `shared/core/pairing/`. Transport in `shared/adapters/transport/` (Ktor WS client). Relay pairing logic in `relay/pairing/`. `RelayServer` in `relay/server/` bridges `ApprovalBridge.outbound: SharedFlow<WireMessage>` to the socket. `relay-cli pair/revoke/serve` surfaces admin ops.

**Tech Stack:** Kotlin 2.1 KMP · kotlinx.{coroutines,serialization,datetime} · **Ktor 3.0 WebSockets** (client + server) · BouncyCastle 1.78+(JVM+Android) · kotlin.test + kotlinx-coroutines-test.

## Patterns to Mirror

### MIRROR-1: Ports & Adapters (project.md §Architecture Patterns)
`shared/core/**` stays framework-free. `TransportPort` sealed interface in `shared/core/transport/`, Ktor-backed impl in `shared/src/jvmMain/.../transport/`. Mirrors W1.4 `ApprovalStore` (port in `relay/approval/`) + `InMemoryApprovalStore` (adapter).

### MIRROR-2: Platform Ed25519 via expect/actual
Rationale in **ADR-0005** + change `add-ed25519-platform-crypto-impl`. Hand-rolling RFC 8032 is audit-heavy and side-channel-risky; BouncyCastle `bcprov-jdk18on` 1.78+ is hardened and Android-API-26+ compatible. iOS ships `NotImplementedError` stub in W1.5; real iOS actual → W2.1 `add-ios-ed25519-actual`.

`shared/src/commonMain/.../Ed25519.kt` declares `expect object Ed25519 { keypair, sign, verify }` + `data class Ed25519KeyPair(secretKey: 64B, publicKey: 32B)`. `secretKey` layout is RFC 8032 §5.1.5 (`seed || pk`). Pure-Kotlin reimpl **prohibited by spec**.

### MIRROR-3: ClientHello framing (spec: `pairing/spec.md` §Ed25519 会话签名)
```jsonc
// First WS frame, Text payload:
{ "op":"hello.client", "deviceId":"...", "sessionId":"S1", "timestampMs":..., "nonce":"b64url(16 rnd)", "sig":"b64url(Ed25519((sessionId||'|'||ts||'|'||nonce).utf8))" }
// Relay response:
{ "op":"hello.ok", "serverTimeMs":..., "protocolVersion":1 }
// or:
{ "op":"hello.err", "reason":"skew|nonce|sig|unpaired|revoked|duplicate-session|malformed|expected-hello" }
```
All join existing `WireMessage` hierarchy (reuses `WireMessageSerializer` + `ProtocolJson` from W1.1). Discriminator `"op"`. Reason alphabet is fixed by `openspec/changes/add-w15-*/specs/pairing/spec.md` → cannot be extended without new change proposal.

### MIRROR-4: Virtual-clock + fake-entropy seams (W1.4 `TimeSeam` pattern)
```kotlin
interface ClockSeam { fun nowMs(): Long }
object SystemClockSeam : ClockSeam { override fun nowMs() = kotlinx.datetime.Clock.System.now().toEpochMilliseconds() }
interface RandomSource { fun nextBytes(size: Int): ByteArray }
expect fun platformSecureRandom(): RandomSource
```
Tests pass `FakeClock(start=1_000_000_000L)` + `FakeRandom(bytesOf(…))`. Same pattern worked cleanly in W1.4.

### MIRROR-5: Nonce cache — bounded, monotonic-time evict
`LinkedHashMap<String, Long>` with access-order LRU + whole-map `@Synchronized`. Spec mandates 10-minute TTL. On each `offer`:purge entries with `nowMs - insertedAt >= ttlMs` then check/insert. Bounded at 10_000 entries. Pattern in `relay/pairing/NonceCache.kt` (Task 7 green).

### MIRROR-6: Single-connection enforcement — token-based (design D1)
`SingleConnectionRegistry.claim(sessionId, token: Long): Boolean` + `release(sessionId, token: Long)`. Second claim → false → `HelloErr("duplicate-session")` + close 1013. Release with wrong token is no-op (fencing). Token source: `AtomicLong.incrementAndGet()` in `ServeCommand`.

### MIRROR-7: Test structure
kotlin.test + JUnit Platform. `runTest` for time-sensitive code. Ktor `testApplication` for end-to-end server tests (no port binding). JVM-only for `relay/` tests; `shared/commonTest/` for KMP-pure modules.

## Complexity / Risk

- **Complexity**: **M** (was H — Ed25519 via BouncyCastle instead of hand-rolled removed ~400 LOC of audit surface).
- **Confidence**: **High** — spec crisp, BouncyCastle test-vector-proven, Ktor WS test harness well-documented.
- **Residual risks**:
  - **Ktor test-host WS close-race**: shutdown before final frame. Mitigate with `incoming.receive()` + explicit client-side `close()`.
  - **iOS Ed25519 stub vs eventual W2.1 actual**: signing/verify never runs on iOS in W1.5 CI — only compile. Mitigated by RFC 8032 vector contract tests that iOS actual MUST pass when it lands.
  - **Live-session revoke NOT implemented** (see NOT-doing). W1.6 change proposal will lift this; current spec (`specs/pairing/spec.md` ADD in add-w15 change) explicitly pins the constrained semantics.

## What this plan is NOT doing

- No SqlDelight persistence of `PubkeyRegistry` — in-memory per-process; W2.3 owns persistence.
- No TLS termination — `ws://` only; W4 packaging adds TLS 1.3.
- No Compose UI for pairing — W2.
- No live-session eviction on `revoke` — deferred to W1.6 with its own change proposal.
- No multi-session fan-out — `SingleConnectionRegistry` is 1 holder per sessionId.
- No E2E mobile test — Maestro/XCUITest arrives in W2.
- No iOS/Android `TransportPort` actual — W1.5 jvmMain only; contract pinned in spec for W2.x.
